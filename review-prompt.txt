I maintain an AI model (SpeciesNet) for classifying wildlife in images from motion-triggered wildlife cameras; the code associated for that model is here:

c:\git\cameratrapai

The model is global, but when a user runs it, in order to avoid predicting species in areas where they clearly don't belong, the user typically specifies the country from which their images come, and if the model predicts a taxon that is not allowed in that country, we walk up the taxonomic tree until we get to a taxon that is allowed.  The rules about what taxa are/aren't allowed in specific locations are hard-coded in a .json file.  We call this process "geofencing".  For example, if the model predicts a European deer in the US, we would check the geofence, see that this species is not allowed, but its genus (cervus) is, so we would output "cervus genus".  If the same prediction happened in Costa Rica, we would see that neither the species nor genus are allowed, but the family is, so we would output "cervidae family".  The rules operate at country level, except in the US, where we have both country- and state-level rules.

The geofencing rules are defined in this .json file:

C:\git\cameratrapai\data\model_package\geofence_release.json

C:\git\cameratrapai\data\model_package\taxonomy_release.txt
